

# フォルダ構成の目次（README）

このリポジトリのルートには、以下の主要フォルダ・ファイルがあります。


【初期状態とフォルダ追加について】
本リポジトリの初期状態では、`data` と `data_definition` の2つのフォルダのみが存在します。

AI対話や演習の進行に応じて、`scripts` や `src`、`output`、`database` などのフォルダは必ず新規に自動作成されます。
Pythonスクリプトやプロジェクト用コードは、必ずこれら新規作成されたフォルダ（`scripts` や `src`）に格納してください。
KPIレポートや分析結果などの成果物は必ず`output`フォルダに保存してください。
SQLファイルやETLツール、SQLite等で作成したデータベースは必ず`database`フォルダに保存してください。
（ルート直下や他の場所にスクリプト・成果物・データベースファイルを置かないようご注意ください）

## data
- 演習用のブロンズ層データ（CSVファイル）が格納されています。
- 各システム（ERP, MES, WMS, TMS, P2Pなど）から吐き出された生データCSVをイメージしてください。
- サブフォルダごとに用途やシステム別に整理されています。

## data_definition
- 演習用ブロンズ層データのテーブル定義書（Markdownファイル）が格納されています。
- 各CSVデータのカラム定義やデータ型、リレーション情報などを記載しています。
- データ設計や分析時の参照用です。

## scripts
- Pythonスクリプト（.pyファイル）を格納するためのフォルダです。
- 新しくスクリプトを作成する場合は、必ずこのフォルダに保存してください。
- データ加工・分析・DB構築などの処理はこのフォルダ内で管理します。

## src
- プロジェクトの規模や目的に応じて、より体系的なコード管理が必要な場合に作成されることがあります。
- モジュール化されたPythonコードや、再利用性の高い処理を格納するためのフォルダです。
- AI対話や開発の進行に応じて追加される場合があります。

## database
- SQLファイル、ETLツール、SQLite等で作成したデータベースファイル（例：.db, .sqlite, .sql など）を格納するためのフォルダです。
- データベース関連の成果物は必ずこの`database`フォルダに保存してください。

## output
- AI対話や演習の進行により作成されるKPIレポートや分析結果ファイル（Markdown, HTML, CSV等）を格納するためのフォルダです。
- 例えば、KPIモニタリングレポートや分析用エクスポートファイルなどは必ずこの`output`フォルダに保存してください。
